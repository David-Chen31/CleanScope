# CleanScope 产品需求冻结文档（PRD-Freeze v1.0）

> 文档性质：**需求冻结（Requirements Freeze）**。本文档一经确认，MVP 阶段的范围、边界、红线即冻结，非经变更评审不得改动。
> 冻结日期：2026-06-14　｜　负责人：产品 & 技术负责人　｜　适用范围：MVP / Beta / v1.0
> 本文档的最高优先级是**划清边界**——明确"做什么、不做什么、绝不做什么"。功能细节服从边界。

---

## 0. 冻结声明与四条不可动摇的产品宪法

以下四条是产品的"宪法条款"，任何版本、任何迭代、任何 KPI 压力下都不得违反。违反即视为产品事故，而非功能缺陷。

1. **AI 不直接删除 C 盘重要文件。** 系统默认只分析、解释、建议；删除动作必须由用户显式触发并经过确认。
2. **解释优先于清理。** 任何"建议处理"前，必须先给出文件来源、归属、依赖关系、风险等级与证据。
3. **用户拥有最终裁决权。** 系统不替用户做删除决定，只提供决策依据。
4. **安全、可解释、可测试、可扩展** 是四个非功能性硬约束，优先级高于任何新功能。

> 任何与上述四条冲突的需求，一律否决，不进入评审。

## 1. 产品定位

**一句话定位：** C 盘文件解释器 + 风险分级助手 + 清理决策支持系统。

**它是什么：**
一个面向真实 Windows 用户的桌面产品，帮助用户在面对 C 盘大文件 / 陌生目录时，搞清楚"这是什么、属于谁、能不能动、怎么动最安全"，并由用户自己做出清理决策。

**它明确不是什么（定位负面清单，防止范围漂移）：**
- ❌ 不是"一键自动清理工具"（不追求"删得多、删得快"）。
- ❌ 不是"系统优化/加速/修复"全家桶。
- ❌ 不是"杀毒/安全防护"软件。
- ❌ 不是"磁盘分区/迁移"工具。
- ❌ 不是"重复文件批量删除器"（v1 不做，见 §6）。

**产品心智锚点：** 用户记住的应该是"**那个能看懂神秘文件夹、而且绝不会让我删错的工具**"，而不是"又一个清理软件"。

## 2. 目标用户

### 2.1 核心用户（MVP 必须打动）
| 优先级 | 用户群 | 核心诉求 | 为何优先 |
|---|---|---|---|
| P0 | **开发者 / 学生** | IDE、Python/Node/Conda/Docker/WSL 缓存爆炸，要安全定位与清理 | 高活跃、高付费意愿、口碑放大器 |
| P0 | **电脑维修 / IT 运维** | 快速归因 C 盘占用、可导出报告 | 最强付费意愿、B 端传播 |

### 2.2 重要用户（Beta 起重点覆盖）
| 优先级 | 用户群 | 核心诉求 |
|---|---|---|
| P1 | **普通进阶用户** | C 盘满、不敢删，要看懂、要安心 |
| P1 | 设计 / 视频 / 游戏玩家 | 素材缓存、更新包、转储文件归因与清理 |

### 2.3 暂不作为目标用户（v1 不为其专门设计）
- 企业 IT 批量终端管理（→ 企业版，路线图后期）。
- 完全不懂文件路径的纯小白（v1 仍假设用户能理解"文件夹/路径"的基本概念；纯小白的"傻瓜模式"留待 v1.x）。

> **裁决：MVP 的一切取舍以"开发者 + 维修人员"是否满意为准绳。** 这两类人满意，产品就有了根据地。

## 3. 核心使用场景（冻结为 5 个，多余场景一律不进 MVP）

| 编号 | 场景 | 用户期望的结果 | 覆盖版本 |
|---|---|---|---|
| S1 | C 盘飘红，扫出 `AppData\Local\Temp` 占 20GB，不知哪些能清 | 分级展示 + 安全清理建议 | MVP |
| S2 | 看到 `Windows\Installer` 一堆随机 `.msi/.msp`，以为是垃圾 | 高风险警示 + 解释为何不能直删 | MVP |
| S3 | 开发者 `.gradle / pip cache / conda pkgs / Docker / WSL .vhdx` 占几十 GB | 归因 + 官方清理/迁移命令 | MVP |
| S4 | 遇到 `ProgramData\Package Cache\{GUID}` 不明文件夹 | AI 解释归属、用途、删除影响 | MVP |
| S5 | 想找 C 盘最大 100 个文件并按"可安全处理程度"排序 | Top N 列表 + 风险排序 | MVP |

> 其他衍生场景（多盘、监控"为什么又满了"、重复文件解释等）一律不属于 MVP 核心场景，归入路线图。

## 4. 版本边界（MVP / Beta / v1.0 正式版）

版本边界是本文档的核心。**每个版本只为达成一个明确目标存在。**

### 4.1 MVP —— 目标：验证"AI 解释 + 风险分级"可行且可信
**定位：** 内部 + 种子用户验证版。不追求好看，追求"解释准、不误判、能用"。

**包含：**
- C 盘大文件 / 大目录扫描，Top N 展示，渐进式结果。
- 路径规则识别 + ≥20 类常见目录知识库（硬规则）。
- 高风险目录识别（黑名单硬规则）。
- 已安装软件列表读取 + 文件元数据/数字签名读取（基础归因）。
- 五级风险分级展示。
- AI 结构化解释（单文件 + 批量），带证据与置信度。
- 辅助操作：打开目录 / 复制路径 / 跳转设置或卸载入口 / 展示推荐命令 / 加入忽略名单。
- Markdown 报告导出。
- **不含任何删除功能**（连"移到回收站"都不做，MVP 纯解释）。

**MVP 退出标准（满足才能进 Beta）：** 见 §9.1。

### 4.2 Beta —— 目标：验证"用户敢用、愿用、信任"
**定位：** 公开测试版。在 MVP 之上加入**受严格保护的有限清理能力**，开始检验真实删除场景的安全性。

**新增：**
- 完整 GUI（首页概览 / 列表页 / 详情页 / 清理建议页 / 安全确认页）。
- **有限清理（首次开放删除）**，但仅限：
  - 仅允许对 **A 级（通常可清理）** 项执行；
  - **默认移入回收站，永不永久删除**；
  - 删除前强制：影响说明 + 二次确认；
  - B/C 级只给"官方清理方式/命令"，不提供直删按钮；
  - **D 级硬禁用删除入口，E 级不给删除建议**。
- 完整操作日志 + 恢复指引。
- 用户反馈闭环（解释对/错、已安全删除、删后出问题、忽略）。
- 进程占用检测、WinGet/Appx 包信息（增强归因）。
- 隐私设置面板（本地/云端模型切换、脱敏开关、隐私提示）。
- HTML / JSON / CSV 报告导出。

**Beta 退出标准：** 见 §9.2。

### 4.3 v1.0 正式版 —— 目标：成熟、可信赖、可商用
**定位：** 面向公众发布、可承载商业模式的成熟产品。

**新增：**
- 稳定性、性能、兼容性达到成熟产品标准（§9.3）。
- 清理前可选**自动备份 / 创建还原点**（在已有回收站之上再加一层）。
- B 级"调用软件官方清理命令"的半自动执行（仍需确认，仍可撤销）。
- 知识库可热更新机制。
- Freemium 商业化能力（免费/Pro 分层，许可与计费）。
- 完善的崩溃上报、遥测（脱敏）、自动更新。

> **版本纪律：** 任何"很想现在就做"的功能，先问它属于哪个版本。不属于当前版本的，写进路线图，不进当前迭代。

## 6. 第一版（截至 v1.0）绝对不做的功能（冻结）

以下功能**在 v1.0 及之前一律不做**，无论用户呼声多高。需求方提出时，统一回复"已列入 v1.x+ 路线图"。

1. **❌ 自动删除 / 一键清理（无需用户逐项确认的批量删除）。** 违反产品宪法。
2. **❌ 永久删除（绕过回收站）。** 永远默认回收站。
3. **❌ 对 D/E 级文件提供任何删除入口。**
4. **❌ 系统修复 / 优化 / 加速 / 注册表清理。** 不是本产品定位。
5. **❌ 驱动级 / 内核级文件占用分析。** 风险与成本过高。
6. **❌ 重复文件批量删除。** v1 最多"解释"，不做"删除"。
7. **❌ 多盘（D/E/F）全面支持。** v1 专注 C 盘（路线图再扩展）。
8. **❌ 企业策略管理 / 批量终端管控。** 留给企业版。
9. **❌ 跨机器 / 云端知识共享同步。**
10. **❌ 自动创建系统还原点作为默认行为**（v1.0 作为"可选项"，更早版本不做）。
11. **❌ 对所有软件缓存的"完整"识别。** 只覆盖高频目录，未知目录一律保守处理。
12. **❌ 联网上传文件内容。** 永远不传内容（见隐私红线）。

## 7. 安全红线（违反任意一条 = 阻断发布，不可妥协）

| 编号 | 红线 | 实现要求 |
|---|---|---|
| SR-1 | **默认不删任何文件** | 无用户显式确认，不执行任何删除 |
| SR-2 | **高风险（D）/ 无法判断（E）禁止删除** | 入口硬禁用/隐藏，代码层面拦截 |
| SR-3 | **删除一律先入回收站** | v1 全程不提供永久删除；绕过回收站的代码路径不存在 |
| SR-4 | **删除前必须双重确认 + 影响说明** | 单击不可删除，必须二次确认并展示后果 |
| SR-5 | **AI 输出必须带证据与置信度** | 无证据/低置信度不得给出"可删"结论 |
| SR-6 | **规则库优先级高于 AI 判断** | 已知高风险目录走硬规则，AI 不得覆盖 |
| SR-7 | **证据不足时只能输出"无法判断"** | 禁止 AI 在证据不足时臆断"可删" |
| SR-8 | **系统关键目录永不建议直删** | Windows/WinSxS/Installer/驱动/系统服务目录硬保护 |
| SR-9 | **所有删除可追溯、可恢复** | 完整操作日志 + 恢复指引 |
| SR-10 | **权限不足时保守处理** | 不强制提权；无权限目录记录而非猜测 |

> 安全红线由自动化测试守护（见 §9 安全验收）。**任一红线测试失败，构建不予发布。**

---

## 8. 隐私红线（违反任意一条 = 阻断发布）

| 编号 | 红线 | 实现要求 |
|---|---|---|
| PR-1 | **绝不上传文件内容** | 任何版本、任何模型都不上传文件正文/二进制内容 |
| PR-2 | **默认本地处理** | 默认本地扫描；云端 AI 为可选、非默认 |
| PR-3 | **上云前强制脱敏** | 用户名、文档名、隐私路径在发送前脱敏 |
| PR-4 | **最小必要信息** | 上云仅含路径模式、扩展名、大小、元数据、哈希摘要等必要字段 |
| PR-5 | **用户可一键关闭云端 AI** | 提供纯本地模式 |
| PR-6 | **云端分析显著告知** | 使用云端前明确提示并需用户知情同意 |
| PR-7 | **遥测可关闭且脱敏** | 任何统计数据脱敏、可选退出（v1.0 引入遥测时生效） |

## 9. 成熟产品验收标准（分版本）

### 9.1 MVP 验收（进 Beta 的门槛）
- 功能：能扫出 C 盘 Top 100 大文件；识别 ≥20 类常见目录；AI 能生成可理解的结构化解释。
- **安全（一票否决）：高风险目录被误判为"可安全删除"的比例 = 0%。**
- 可解释：每条结论都附证据与置信度；低置信度正确输出"无法判断"。
- 价值：真实测试机上"可安全释放空间"的识别量 ≥ 10GB。
- 可用：种子用户（开发者/维修）能据解释独立做出判断。

### 9.2 Beta 验收（进正式版的门槛）
- 安全红线 SR-1～SR-10 **全部有自动化测试覆盖且通过**。
- 有限删除全流程经真实用户测试，**零误删事故**（删除均可在回收站恢复）。
- 归因准确率、风险分级准确率达到设定阈值（具体阈值在评估阶段标定）。
- 用户对建议的信任度 / 采纳率达到设定目标（NPS、留存为参考）。
- 隐私红线 PR-1～PR-6 全部满足并经审查。

### 9.3 v1.0 正式版验收（成熟产品标准）
| 维度 | 标准 |
|---|---|
| 安全 | 全部安全/隐私红线测试通过；红线为发布门禁 |
| 稳定性 | 长时间扫描无崩溃；崩溃率低于设定阈值；异常可恢复 |
| 性能 | 百万级文件扫描在可接受时间内完成；扫描可中断/恢复；内存占用受控；AI 批量解释有缓存 |
| 兼容性 | Windows 10 / 11、NTFS、普通+管理员权限均验证通过 |
| 可恢复 | 删除可回收站恢复；v1.0 可选备份/还原点；完整操作日志 |
| 可维护 | 知识库可更新；规则可扩展；具备崩溃上报与（脱敏）遥测 |
| 商业化 | 免费/Pro 分层、许可与计费链路通畅；无捆绑、无广告、无偷删 |
| 体验 | 普通进阶用户能在无人指导下完成"扫描→看懂→决策"闭环 |

## 10. 后续开发任务总路线图

> 路线图给出**阶段目标与里程碑**，不含实现细节。每阶段以"退出标准"驱动，达标才进入下一阶段。

### 阶段 0：需求冻结（本阶段）
- 产出：本需求冻结文档 + 风险分级体系 + UI 草图 + 技术选型确认。
- 退出：本文档评审通过、四条宪法与红线确认。

### 阶段 1：MVP —— 扫描 + 规则 + 风险分级
- 大文件扫描、Top N、目录聚合、缓存。
- 第一版知识库（≥20 类）+ 高风险黑名单 + 风险评分。
- 退出标准：§9.1。

### 阶段 2：MVP —— 归因 + AI 解释
- 基础归因（已安装软件 + 元数据 + 数字签名）。
- AI 结构化解释、防幻觉约束、单/批解释。
- 退出标准：解释可理解、证据完整、红线 SR-5/6/7 初步达成。

### 阶段 3：Beta —— GUI + 有限清理 + 安全机制
- 完整界面五页 + 安全确认流。
- A 级有限清理（回收站 + 双确认）+ 操作日志 + 恢复指引。
- 进程占用 / WinGet / Appx 增强归因。
- 隐私设置面板。
- 退出标准：§9.2，安全红线全测试通过。

### 阶段 4：Beta —— 评估与信任建设
- 测试样本集 + 人工标注；归因/风险分级准确率标定。
- 误判案例库；用户反馈闭环上线。
- 退出标准：准确率与信任指标达标。

### 阶段 5：v1.0 —— 成熟化与商业化
- 性能/稳定性/兼容性加固；备份/还原点；官方清理命令半自动执行。
- 知识库热更新；崩溃上报与脱敏遥测；自动更新。
- Freemium 商业化链路。
- 退出标准：§9.3。

### 阶段 6+：路线图（v1.x 及以后，明确不在 v1 范围）
多盘支持 → 缓存迁移助手 → 本地小模型 → 社区知识库 → 软件专属清理插件 → 重复文件解释 → 企业策略/批量 → "为什么 C 盘又满了"长期监控（从工具进化为磁盘健康管家）。
