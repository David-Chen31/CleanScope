# CleanScope 项目计划书

> **看得懂、删得安心的 C 盘助手**
> 不替你删，先帮你看懂——每一个大文件"是什么、属于谁、删了会怎样、怎么处理最安全"。

---

## 0. 一句话定位（电梯演讲）

> 市面上的清理工具只会告诉你"这里占了 20GB"，然后让你赌一把要不要删。
> **CleanScope 告诉你这 20GB 到底是什么、能不能动、怎么动最安全，让清理从"提心吊胆地赌"变成"看着证据做决定"。**

适合放在官网首屏、应用商店截图首图、答辩第一页 PPT。

## 一、为什么做这个产品（痛点 + 机会）

### 1.1 用户的真实痛点

几乎每个 Windows 用户都经历过：C 盘飘红 → 打开某清理软件 → 扫出一堆 `{8F3A...}` 的目录和 18GB 的 `Windows\Installer` → **不敢删，也不知道能不能删** → 要么放着不管，要么硬删然后某个软件打不开。

现有工具的共同短板可以归为一句话：**它们会"找垃圾"，但不会"解释文件"。** 用户缺的不是"扫描能力"，而是面对陌生文件时的**判断依据和安全感**。

### 1.2 市场机会

- Windows 桌面用户基数巨大，"C 盘满了怎么办"是常年高频搜索词。
- 现有清理软件普遍存在**信任危机**：捆绑广告、夸大扫描结果、误删导致系统问题，用户既离不开又不敢信。
- AI（大模型）第一次让"用大白话解释一个陌生系统文件"在工程上变得可行——这是过去十年清理软件做不到、而现在恰好能做的事。

**结论：这是一个"老需求 + 新能力"的窗口。** 谁能第一个把"可解释、不删错、值得信任"做成产品心智，谁就能在一个被用户既需要又厌恶的赛道里建立差异化。

## 三、核心价值主张（说人话的客户收益）

把"功能"翻译成"用户得到什么"：

1. **多腾出几十 GB，而且不会删错。** 看着证据清理，系统不崩、软件不坏。
2. **看懂那串看不懂的文件夹。** 随机字符串、GUID、缓存目录，一键问"这是啥"。
3. **省去搜索和试错。** 不用再去贴吧、知乎搜"xxx 文件夹能删吗"，答案直接带证据给你。
4. **给出更聪明的处理方式。** 很多空间不用删也能省——改缓存路径、用软件自带清理、移到别的盘，CleanScope 会优先推荐。
5. **全程你说了算。** AI 只解释和建议，删不删、怎么删，永远是你点的。

> 营销主张：**"清理 C 盘，不再靠运气。"**

## 六、产品定位与不可动摇的原则

**定位：C 盘文件解释器 + 风险分级助手 + 清理决策支持系统**（不是"自动清理器"）。

四条原则既是产品底线，也是营销资产：

1. **默认不删。** 未经明确确认绝不执行删除。
2. **证据优先，结论透明。** 每条判断都附路径、签名、安装记录、进程占用等证据。
3. **风险分级，而非能删/不能删。** 五级体系贴合真实复杂度。
4. **优先推荐"不删也能省"的安全路径。** 官方清理、改路径、迁移优先于删除。

> 这四条不只是工程约束，更是我们对抗"清理软件 = 不靠谱"行业印象的**信任牌**。把它们写进官网、写进应用商店描述。

## 七、安全与信任：把"不删错"做成卖点

清理软件的真正生死线是**信任**。我们把安全设计当成最大的营销点来做：

- **默认移入回收站**，永不直接永久删除；高风险项默认隐藏删除入口。
- **删除前影响说明 + 二次确认**，告诉你"删了之后某软件会怎样"。
- **AI 必带证据和置信度**，区分"事实证据"和"AI 推测"，低置信度直接说"无法判断，建议谨慎"，绝不嘴硬。
- **规则库优先级高于 AI 猜测**，已知高风险目录走硬规则，杜绝大模型幻觉造成误判。
- **隐私本地优先**：默认本地扫描，不上传文件内容；上云分析前对用户名/文档名等敏感路径脱敏；可一键关闭云端 AI、切换本地模型，并显著提示。
- **完整操作日志 + 恢复指引**，出问题能追溯、能还原。

> 对外口号：**"我们宁可少删，也绝不删错。"**

## 八、产品功能（精简版）

> 完整字段与交互见附录；此处只列对客户和评审最关键的。

### 8.1 扫描
一键扫 C 盘大文件/大目录，按大小/类型/时间排序，渐进式展示（先出结果再继续），自动处理无权限目录，支持普通/管理员两种模式。

### 8.2 归因（这个文件属于谁）
多证据融合：路径规则 + 已安装软件/卸载注册表 + WinGet/Appx 包信息 + 文件元数据与数字签名 + 进程占用 + 常见目录知识库 + 扩展名模式 + AI 语义补充。**输出候选列表 + 置信度**，而非武断一个答案。

### 8.3 五级风险分级

| 等级 | 含义 | 建议 |
|---|---|---|
| A 通常可清理 | 临时文件、明确缓存、可再生成 | 可清理，仍需确认 |
| B 走官方方式清理 | 软件/包/浏览器缓存 | 用软件设置或命令清理 |
| C 谨慎处理 | 应用数据、旧版本、日志、备份 | 先备份/确认用途 |
| D 高风险 | 系统组件、安装器缓存、驱动、核心依赖 | 不建议直接删 |
| E 无法判断 | 证据不足 | 不给删除建议，提示进一步确认 |

### 8.4 AI 结构化解释
每个文件输出：是什么 / 可能属于谁 / 为什么在这 / 删了会怎样 / 推荐怎么处理 / 判断依据 / 置信度 / 是否需进一步确认。支持追问（"为什么不建议删？""有没有更安全的办法？"）。

### 8.5 决策与操作（MVP 只做辅助、不代删）
打开所在目录 / 复制路径 / 跳转 Windows 设置或软件卸载入口 / 展示推荐命令 / 加入忽略名单 / 导出报告。
后续版本（严格确认 + 回收站 + 备份）才谨慎开放有限清理。

## 十、推广与增长策略（原文缺失，重点补充）

### 10.1 增长飞轮
**好用 → 用户分享报告/截图 → 更多人知道"原来能看懂神秘文件" → 用户贡献反馈完善知识库 → 解释更准 → 更好用。**

### 10.2 三条具体打法
1. **内容种草（低成本起量）**：做一系列"C 盘里这些文件夹到底能不能删"的短视频/图文（`Windows\Installer`、`Package Cache`、`.vhdx`、`pip cache`……）——这些都是高搜索量长尾词，CleanScope 是天然答案，引导下载。
2. **开发者社区口碑**：在 GitHub、V2EX、掘金、Reddit r/Windows 等投放"开发者 C 盘救星"叙事，开源部分规则库吸引贡献。
3. **维修/装机渠道**：技术版做成"上门排查神器"，一张专业报告 = 维修师傅的专业背书，自带 B 端传播。

### 10.3 可传播的产品设计
- **报告自带品牌水印**，客户转发即推广。
- **"我帮你省了 X GB / 避免了一次事故"** 的成就感反馈，鼓励截图分享。
- **社区知识库**：用户反馈"这个目录是 XX 软件的"可沉淀进公共知识库，越用越准——既是增长飞轮也是护城河。

## 十一、护城河：会自我增强的清理知识库

竞品可以抄 UI，难抄的是**积累出来的、被真实用户验证过的目录知识库**。

- 初版覆盖 20+ 类高频目录：系统目录、Windows Installer/WinSxS/更新缓存、浏览器、VS/JetBrains/VS Code、npm/pnpm/yarn、pip/conda/uv、Maven/Gradle/NuGet、Docker/WSL、Steam/Epic、日志、转储、镜像、云盘同步……
- 每条规则带：匹配模式、类别、风险等级、能否直删、说明、**推荐的安全处理方式**、证据类型、置信度。
- **用户反馈闭环**（解释对/错、我知道它属于谁、已安全删除、删后出问题、忽略）持续校准规则与风险模型。
- 知识库越大越准 → 解释越好 → 用户越多 → 反馈越多 → 知识库越大。**时间是我们的朋友。**

## 十二、技术架构（精简）

六层架构：**表现层 → 扫描层 → 系统信息采集层 → 规则与知识库层 → AI 推理层 → 操作执行层。**

- **扫描层**：递归遍历、最小堆维护 Top N、目录自底向上聚合、跳过无权限目录、结果持久化缓存。
- **采集层**：路径/扩展名/元数据/数字签名/已安装应用/卸载注册表/WinGet/Appx/进程占用/目录规则库。
- **规则库层**：硬规则优先，覆盖高风险黑名单与安全清理映射表。
- **AI 层**：结构化"证据摘要 in → 结构化解释 out"，强约束防幻觉（详见附录 Prompt）。
- **执行层**：MVP 仅辅助操作，后续才在多重保护下谨慎开放清理。

**技术选型推荐（MVP）：Python + PySide6 + SQLite + 规则库 + LLM API。**
理由：快速验证"AI 解释文件用途与风险"这一核心假设；扫描、规则、AI 调用都易实现。产品化阶段再迁移到 **Tauri（性能/体积/安全）** 或 **.NET WinUI（深度 Windows 集成）**。

## 十三、MVP 范围与成功标准

**MVP 必做：** 大文件/大目录扫描、Top N 展示、路径规则识别、高风险目录识别、常见缓存识别、已安装软件读取、元数据读取、AI 解释、风险分级、Markdown 报告导出。

**MVP 不做：** 自动删除、系统修复、驱动级占用分析、云端同步清理、跨机知识共享、企业策略、自动还原点。

**MVP 成功标准（既是验收也是 PMF 信号）：**
1. 能找出 C 盘 Top 100 大文件；
2. 识别 ≥20 类常见目录；
3. **高风险目录零误判为"可安全删除"**（安全红线，不可破）；
4. 普通用户能看懂解释并独立做出判断；
5. 能导出完整报告；
6. 真实测试中平均"可安全释放空间"识别 ≥ 10GB（量化价值）。

## 十四、北极星指标与评估（原文指标 + 升级）

**北极星指标：用户经 CleanScope 安全释放的总空间（GB），且零误删事故。**
它同时绑定了"有用"（释放空间）和"可信"（零事故）。

| 类别 | 关键指标 |
|---|---|
| 价值 | 平均可释放空间、用户实际释放空间、报告导出率 |
| **安全（最高优先级）** | 高风险误判为可删比例（目标 ≈0）、AI 危险建议比例、删除建议是否带替代方案 |
| 信任/留存 | 用户对建议的信任度、采纳率、次月留存、NPS |
| 质量 | 归因准确率、风险分级准确率、解释可理解性评分 |
| 性能 | 百万文件扫描耗时、内存占用、AI 批量解释耗时、缓存命中率 |

## 十五、风险与应对

| 8 | 收尾/展示 | 完整文档、**真机演示视频**、答辩 PPT、开源仓库 |

> 建议把第 8 周的"真机演示视频"提前到第 6 周就开始拍——它是最好的推广素材，也是最好的进度检验。

## 十七、后续路线图

近期：多盘（D/E/F）、缓存迁移助手、本地小模型推理、社区知识库。
中期：软件专属清理插件、重复文件解释、企业策略白/黑名单、清理前自动备份/还原点。
远期：**"为什么我的 C 盘又满了"长期监控**——从一次性清理工具，进化为常驻的磁盘健康管家（提升留存与订阅价值）。

## 十八、结语

C 盘清理是个"人人都需要、人人都不放心"的市场。传统工具卡在"能找垃圾、不会解释"，而大模型恰好补上了"用大白话解释陌生系统文件"这块最后的拼图。

**CleanScope 的赌注是：在一个被信任问题拖累的赛道里，把"看得懂、不删错、值得信"做成产品心智。**
我们不比谁删得快，我们让用户第一次能**看着证据、放心地**清理自己的电脑。

> **清理 C 盘，不再靠运气。**

## 附录 A：AI 输入/输出与 Prompt 约束

**结构化输入示例**
```json
{
  "path": "C:\\ProgramData\\Package Cache\\{GUID}",
  "size": "4.2GB", "extension": "", "metadata": {},
  "matched_rules": ["Package Cache", "GUID directory"],
  "related_apps": ["Visual Studio"],
  "process_using": [], "known_risk": "medium", "confidence": 0.72
}
```

**结构化输出示例**
```json
{
  "what_is_it": "可能是某软件安装包缓存",
  "owner_app": "Visual Studio 或相关安装器",
  "risk_level": "B", "can_delete_directly": false,
  "recommended_action": "使用官方安装器或缓存管理方式处理",
  "reasoning": ["路径位于 ProgramData Package Cache", "目录名是 GUID", "与已安装应用匹配"],
  "confidence": 0.72,
  "user_friendly_explanation": "..."
}
```

**Prompt 约束**
1. 不得在证据不足时断言可删；2. 不得建议直删系统关键目录；3. 不把"可能"说成"确定"；4. 必须输出风险等级与置信度；5. 必须说明判断依据与删除影响；6. 必须优先推荐官方清理方式；7. 低置信度提示谨慎并允许"无法判断"。

## 附录 B：风险评分映射（0–100，越高越不建议删）
0–20 通常可清理 / 21–40 走官方方式 / 41–60 谨慎 / 61–80 高风险 / 81–100 不建议删或无法判断。
影响因素：系统目录(+)、临时目录(−)、安装器缓存(+)、浏览器缓存(−)、被进程占用(+)、可再生成(−)、用户文档(+)、有官方清理方式(走官方)、证据不足(不确定性+)。
